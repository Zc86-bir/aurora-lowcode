## 二、全局关键数据流

### 2.1 租户 ID 透传链路（JWT → DB 拦截器）

```
[HTTP Request]
    │
    ├── Header: X-Tenant-Id: UUID
    │   └── 或使用 JWT Token 中的 tenant_id claim
    │
    ▼
[ApiGatewayController]
    │ @RequestHeader("X-Tenant-Id") UUID tenantId
    │ 或从 JWT 解析: JwtAuthenticationFilter.extractTenantId(token)
    │
    ▼
[TenantContext.setContext(tenantId, userId)]
    │ ThreadLocal / ScopedValue 存储
    │ TenantInfo: { tenantId, tenantCode, tier, quota, status }
    │
    ▼
[Application Layer]
    │ TenantContext.getCurrentTenantId() 获取租户上下文
    │ 所有 UseCase 方法签名包含 tenantId 参数
    │
    ▼
[Repository / Infrastructure]
    │ ├── Schema 隔离: Connection.setSchema("tenant_" + tenantId)
    │ ├── RLS 模式: Hibernate @Filter + SET LOCAL tenant.id = ?
    │ └── Database 隔离: DataSource 路由 → TenantRoutingDataSource
    │
    ▼
[SQL 执行]
    │ 所有查询强制 WHERE tenant_id = ?
    │ DDL 操作限定在当前租户 Schema
    │
    ▼
[PostgreSQL]
    │ tenant_abc123.form_metadata
    │ tenant_abc123.report_metadata
    │ ...
```

**关键文件**：
- `TenantContext.java` — 租户上下文接口
- `TenantLifecycleManager.java` — 租户 Schema 创建/路由/删除
- `MultiTenantDataSourceManager.java` — 数据源路由
- `ApiGatewayController.java` — 从 Header 提取 tenantId

### 2.4 元数据热加载数据流

```
[数据库: metadata 表]
    │
    ▼
[MetadataHotReloadManager.hasChanges(tenantId, name)]
    │ 1. findByTenantAndName(tenantId, name)
    │ 2. 计算 SHA-256 checksum
    │ 3. 对比 ConcurrentHashMap 中的缓存
    │
    ▼
[checksum 不同?]
    │
    ├──► 是: MetadataSnapshot(metadataId, checksum, version, status, lastLoadedAt)
    │    │ 原子替换缓存
    │    │ 计算 MetadataDiff (version/status/content)
    │    │ 通知所有 ReloadListener
    │
    └──► 否: 使用缓存 (无锁读)
    │
    ▼
[回滚: rollback(tenantId, name, targetVersion)]
    │ 1. findByIdAndVersion(id, targetVersion)
    │ 2. 验证版本一致性
    │ 3. metadataRepository.rollback(id, targetVersion)
    │ 4. 更新缓存 checksum
```

**关键文件**：`MetadataHotReloadManager.java`、`MetadataRepository.java`

### 2.5 不可篡改审计数据流

```
[业务操作: 创建/更新/删除/权限检查/Skill执行]
    │
    ▼
[ImmutableAuditLogger.append(AuditEntry)]
    │ 1. 读取 chainTipHash (当前链尾 hash)
    │ 2. 计算: newHash = SHA256(chainTipHash + tenantId + userId + action + ...)
    │ 3. 创建: AuditChainEntry(seqNum, entry, prevHash, newHash, timestamp)
    │ 4. 追加到 ConcurrentLinkedQueue
    │ 5. 更新 chainTipHash = newHash
    │
    ▼
[完整性验证: verifyChain()]
    │ 1. 从 GENESIS_HASH 开始遍历
    │ 2. 对每个 entry 重新计算 hash
    │ 3. 对比 stored hash vs computed hash
    │ 4. 不匹配 → ChainVerificationResult(false, ...)
    │ 5. 全部匹配 → ChainVerificationResult(true, ...)
    │
    ▼
[导出: export(from, to)]
    │ 过滤时间范围内的 entries
    │ 计算 exportHash = SHA256(concat(all entry hashes))
    │ 输出: ExportResult(entries, exportHash, count, from, to, exportedAt)
```

**关键文件**：`ImmutableAuditLogger.java`
